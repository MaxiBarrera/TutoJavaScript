JavaScript Let

Anterior Proximo

La palabra clave let se introdujo en ES6 (2015)

Las variables declaradas con let tienen un alcance de bloque

Las variables declaradas con let deben declararse antes de su uso

Las variables declaradas con let no se pueden volver a declarar en el mismo ámbito


Ámbito de aplicación del bloque

Antes de ES6 (2015), JavaScript no tenía ámbito de bloque.

JavaScript tenía alcance global y alcance de función.

ES6 introdujo dos nuevas palabras clave de JavaScript: let y const

Estas dos palabras clave proporcionan el alcance del bloque en JavaScript:


Ejemplo

No se puede acceder a las variables declaradas dentro de un bloque { } desde fuera del bloque:


{
  let x = 2;
}
// x can NOT be used here

Alcance global

Las variables declaradas con var siempre tienen alcance global.

Las variables declaradas con la palabra clave var NO pueden tener alcance de bloque:


Ejemplo

Se puede acceder a las variables declaradas var dentro de un bloque { } desde fuera del bloque:


{
  var x = 2;
}
// x CAN be used here

No se puede redeclarar

Las variables definidas con let no se pueden redeclarar.

No se puede redeclarar accidentalmente una variable declarada con let.


Con let usted no puede hacer esto:


let x = "John Doe";

let x = 0;

Las variables definidas con var se pueden redeclarar.


Con var usted puede hacer esto:


var x = "John Doe";

var x = 0;

Redeclaración de variables

Redeclarar una variable utilizando la palabra clave var puede ocasionar problemas.

Al redeclarar una variable dentro de un bloque también se redeclarará la variable fuera del bloque:


Ejemplo

var x = 10;
// Here x is 10

{
var x = 2;
// Here x is 2
}

// Here x is 2
Check!

Redeclarar una variable usando la palabra clave let puede resolver este problema.

Redeclarar una variable dentro de un bloque no redeclarará la variable fuera del bloque:


Ejemplo

let x = 10;
// Here x is 10

{
let x = 2;
// Here x is 2
}

// Here x is 10
Check!

Diferencia entre var, let y const


Alcance Redeclarar Reasignar Hoisted Vincula esto
var no si si si si
let si no si no no
const si no no no no

¿Qué es bueno?

let y const tienen alcance de bloque.

let y const no puede ser redeclarado.

let y const debe declararse antes de su uso.

let y const no se vincula a this

let y const no se izan

¿Qué no está bien?

var no es necesario declararlo

var se iza

var se vincula a esto

Compatibilidad con navegadores

Redeclarando

Se permite redeclarar una variable de JavaScript var en cualquier parte de un programa:


Ejemplo

var x = 2;
// Now x is 2

var x = 3;
// Now x is 3
Check!

Con let, NO se permite volver a declarar una variable en el mismo bloque:


Ejemplo

var x = 2;   // Allowed
let x = 3;   // Not allowed

{
let x = 2;   // Allowed
let x = 3;   // Not allowed
}

{
let x = 2;   // Allowed
var x = 3;   // Not allowed
}

Se permite redeclarar una variable con let, en otro bloque:


Ejemplo

let x = 2;   // Allowed

{
let x = 3;   // Allowed
}

{
let x = 4;    // Allowed
}
Check!

Dejar izar

Las variables definidas con var se elevan a la parte superior y se pueden inicializar en cualquier momento.

Significado: Puedes utilizar la variable antes de que se declare:


Ejemplo

Esto está bien:

carName = "Volvo";
var carName;
Check!

Las variables definidas con let también se elevan a la parte superior del bloque, pero no se inicializan.

Significado: Usar una variable let antes de declararla dará como resultado ReferenceError:


Ejemplo

carName = "Saab";
let carName = "Volvo";
Check!
