JavaScript Strings

Anterior Proximo

Las cadenas sirven para almacenar texto.

Las cadenas se escriben entre comillas.


Uso de comillas

Una cadena de JavaScript es cero o más caracteres escritos entre comillas.


Ejemplo

let text = "John Doe";
Check!

Puedes utilizar comillas simples o dobles:


Ejemplo

let carName1 = "Volvo XC60";  // Double quotes  
let carName2 = 'Volvo XC60';  // Single quotes
Check!

Nota:

Las cadenas creadas con comillas simples o dobles funcionan de la misma manera.

No hay diferencia entre los dos.


Citas dentro de citas

Puedes utilizar comillas dentro de una cadena, siempre que no coincidan con las comillas que rodean la cadena:


Ejemplo

let answer1 = "It's alright";
let answer2 = "He is called 'Johnny'";
let answer3 = 'He is called "Johnny"';
Check!

Cadenas de plantillas

Las plantillas se introdujeron con ES6 (JavaScript 2016).

Las plantillas son cadenas encerradas entre comillas simples (`Esta es una cadena de plantilla`).

Las plantillas permiten comillas simples y dobles dentro de una cadena:


Ejemplo

let text = `He's often called "Johnny"`;
Check!

Nota:

Las plantillas no son compatibles con Internet Explorer.

Longitud de la string

Para encontrar la longitud de una cadena, utilice la propiedad incorporada length:


Ejemplo

let text = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";
let length = text.length;
Check!

Personajes de escape

Dado que las cadenas deben escribirse entre comillas, JavaScript malinterpretará esta cadena:

let text = "We are the so-called "Vikings" from the north.";

La cadena se cortará a "Somos los llamados".

Para resolver este problema, puedes utilizar un carácter de escape de barra invertida.

El carácter de escape de barra invertida (\) convierte los caracteres especiales en caracteres de cadena:

Code Result Description
\' ' Comilla simple
\" " Comillas dobles
\\ \ Barra invertida

Ejemplos

\" inserta una comilla doble en una cadena:


let text = "We are the so-called \"Vikings\" from the north.";
Check!

\' inserta una comilla simple en una cadena:


let text= 'It\'s alright.';
Check!

\\ inserta una barra invertida en una cadena:


let text = "The character \\ is called backslash.";
Check!

Hay otras seis secuencias de escape válidas en JavaScript:

Code Result
\b Retroceso
\f Feed de formulario
\n Nueva linea
\r Retorno de carro
\t Pestaña horizontal
\v Tabulador vertical

Nota:

Los 6 caracteres de escape que aparecen arriba se diseñaron originalmente para controlar máquinas de escribir, teletipos y faxes. No tienen ningún sentido en HTML.

Rompiendo las largas lineas

Para facilitar la lectura, los programadores a menudo prefieren evitar líneas de código largas.

Una forma segura de dividir una declaración es después de un operador:


Ejemplo

document.getElementById("demo").innerHTML =
"Hello Dolly!";
Check!

Una forma segura de dividir una cadena es mediante la suma de cadenas:


Ejemplo

document.getElementById("demo").innerHTML = "Hello " +
"Dolly!";
Check!

Cadenas de plantillas

Las plantillas se introdujeron con ES6 (JavaScript 2016).

Las plantillas son cadenas encerradas entre comillas simples (`Esta es una cadena de plantilla`).

Las plantillas permiten cadenas multilínea:


Ejemplo

let text =
`The quick
brown fox
jumps over
the lazy dog`;
Check!

Nota:

Las plantillas no son compatibles con Internet Explorer.

Cadenas de JavaScript como objetos

Normalmente, las cadenas de JavaScript son valores primitivos, creados a partir de literales:


let x = "John";

Pero las cadenas también se pueden definir como objetos con la palabra clave new:


let y = new String("John");

Ejemplo

let x = "John";
let y = new String("John");
Check!

No cree objetos String.

La palabra clave new complica el código y ralentiza la velocidad de ejecución.

Los objetos de cadena pueden producir resultados inesperados:


Al utilizar el operador ==, x e y son iguales:


let x = "John";
let y = new String("John");
Check!

Tenga en cuenta la diferencia entre (x==y) y (x===y).


(x == y)¿Verdadero o falso?


let x = new String("John");
let y = new String("John");
Check!

(x === y)¿Verdadero o falso?


let x = new String("John");
let y = new String("John");
Check!

Comparar dos objetos JavaScript siempre devuelve falso.
